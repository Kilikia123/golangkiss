# golangkiss

Решение для ДЗ 2.

## Что сделано

- Postgres подключен через GORM.
- В БД хранится вся модель поста, кроме лайков.
- Redis используется для:
  - общего числа лайков поста;
  - флага `liked_by_me` для конкретного пользователя;
  - кеша главной страницы для `offset=0, limit=10`.
- При создании, обновлении и удалении поста кеш главной страницы инвалидируется.

## Выбор структур Redis

- Общее число лайков: `string` по ключу `post:{id}:likes_count`.
- Кто лайкнул пост: `set` по ключу `post:{id}:liked_users`.

Почему так:

- `INCR` и `DECR` хорошо подходят для счетчика лайков.
- `SISMEMBER` удобно отвечает на вопрос, лайкнул ли конкретный пользователь пост.
- `SADD`/`SREM` защищают от дублей лайков и повторных анлайков.

## Оптимизация кеша главной страницы

В кеш Redis кладется только сама лента постов без персонализированного флага лайка.

При чтении главной страницы:

1. посты берутся из кеша;
2. затем одним pipeline в Redis достаются:
   - счетчики лайков;
   - флаги `liked_by_me` для текущего пользователя.

Это позволяет не делать 20 отдельных сетевых round-trip на 10 постов, а получить данные за один батч-запрос.

```bash
export POSTGRES_DSN='host=localhost port=5432 user=postgres password=postgres dbname=blog sslmode=disable'
export REDIS_ADDR='localhost:6379'
export REDIS_DB='0'
export REDIS_PASSWORD=''
export GRPC_ADDR=':8080'
export HTTP_ADDR=':8090'
```
